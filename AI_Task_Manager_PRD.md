## 1. Обзор продукта

**AI Task Manager** — интеллектуальная система управления задачами с встроенным ИИ-ассистентом на базе Claude (Anthropic). Продукт предназначен для персонального и командного использования, обеспечивает полный цикл работы с задачами: создание, назначение исполнителей, контроль сроков и автоматические email-уведомления.

> **Ключевая цель продукта:**
> Создать адаптивный (mobile-first) таск-менеджер с ИИ-ассистентом, который автоматически управляет задачами, уведомлениями и коммуникацией с исполнителями — без ручных напоминаний.

## 2. Проблема и контекст

### 2.1 Описание проблемы

- Менеджеры и руководители тратят значительное время на ручное отслеживание задач
- Исполнители забывают об обновлении статусов и сроках
- Нет единой точки уведомлений: email, мессенджеры, календари не связаны
- Мобильные интерфейсы существующих решений перегружены и неудобны

### 2.2 Целевая аудитория

| Сегмент | Описание | Ключевая потребность |
|---|---|---|
| Руководитель | Ставит задачи, контролирует сроки | Автоматические напоминания и статусы |
| Менеджер проекта | Распределяет задачи по команде | Назначение и отслеживание исполнителей |
| Исполнитель | Получает и выполняет задачи | Чёткие уведомления о дедлайнах |
| Индивидуал | Управляет личными задачами | Простой интерфейс, ИИ-помощник |

## 3. Функциональные требования

### 3.1 Поля задачи (обязательные блоки)

| Поле | Тип | Обязательно | Описание |
|---|---|---|---|
| Наименование задачи | Текст (textarea) | Да | Полное описание задачи |
| Дата исполнения | Date picker | Да | Дедлайн выполнения |
| Приоритет | Select (3 уровня) | Нет | Обычный / Высокий / Критичный |
| Статус исполнения | Pills-выбор | Да | Новая / В работе / Выполнена / Отложена |
| Email владельца | Email input | Да | Адрес для получения уведомлений |
| Исполнитель (ФИО) | Text input | Условно | Назначается опционально |
| Email исполнителя | Email input | Условно | Если исполнитель назначен |
| Контактный телефон | Tel input | Нет | Телефон исполнителя |

### 3.2 Назначение исполнителя

При сохранении задачи система **обязана** задать вопрос о назначении исполнителя:

- Отображается toggle-переключатель «Назначить исполнителя?»
- При активации отображаются поля: ФИО, Email, Контактный телефон
- Email исполнителя становится обязательным при включённом toggle
- При сохранении исполнителю автоматически направляется уведомление

### 3.3 Система уведомлений

| Событие | Получатель | Содержание |
|---|---|---|
| При сохранении задачи | Владелец | Подтверждение создания + детали |
| При сохранении задачи | Исполнитель | Информация о назначенной задаче |
| За 1 день до дедлайна | Владелец | Напоминание о приближении срока |
| В день дедлайна | Владелец | Уведомление о наступлении срока |
| За 1 день до дедлайна | Исполнитель | Напоминание отправить статус владельцу |

### 3.4 Интеграция с календарём

- При создании задачи автоматически формируется событие в календаре
- Поддержка форматов: Google Calendar (iCal / .ics), Outlook Calendar
- Событие содержит: название задачи, дату, исполнителя, приоритет
- Напоминания синхронизируются с настройками уведомлений

### 3.5 ИИ-ассистент (чат)

Встроенный чат на базе Claude API (`claude-sonnet-4-20250514`) выполняет:

- Распознавание задач из свободного текста
  - Пример: _"Согласовать договор с Ивановым до 15 июня"_ → автозаполнение формы
- Ответы на вопросы о текущих задачах и дедлайнах
- Контекстные подсказки по управлению задачами
- Уведомления об изменениях статусов в чате

## 4. Нефункциональные требования

### 4.1 Дизайн и адаптивность

| Требование | Описание |
|---|---|
| Mobile-first | Основной breakpoint — 480px, оптимизировано под смартфоны |
| Адаптивность | Поддержка экранов от 320px до 1920px |
| Dark mode | Автоматическое переключение через CSS-переменные |
| Иконки | Tabler Icons (outline), без кастомных SVG-путей |
| Шрифты | Системный sans-serif, fallback Arial |
| Цвета | CSS-переменные Anthropic Design System |
| Анимации | Минимальные: transition 0.15s на border/background |

### 4.2 Производительность

- Время отклика интерфейса: < 100 мс для локальных операций
- Ответ ИИ-ассистента: < 3 сек (streaming)
- Сохранение задачи: < 500 мс
- Email-уведомление: < 5 сек после сохранения

### 4.3 Безопасность

- Email-адреса хранятся в зашифрованном виде
- API-ключи не передаются на клиент
- HTTPS для всех запросов
- Валидация email-форматов на клиенте и сервере

## 5. Пользовательский поток (UX Flow)

### 5.1 Создание задачи через форму

| # | Шаг | Описание |
|---|---|---|
| 1 | Открытие формы | Пользователь переходит на вкладку «Задача» |
| 2 | Заполнение полей | Название, дата, приоритет, статус, свой email |
| 3 | Вопрос об исполнителе | Система показывает toggle «Назначить исполнителя?» |
| 4 | Данные исполнителя | Если toggle ON — ввод ФИО, email, телефона |
| 5 | Сохранение | Нажатие кнопки «Сохранить задачу» |
| 6 | Валидация | Проверка обязательных полей, показ ошибок |
| 7 | Уведомления | Email владельцу + email исполнителю (если назначен) |
| 8 | Календарь | Автоматическое создание события в календаре |
| 9 | Редирект | Переход на вкладку «Список» с новой задачей |

### 5.2 Создание задачи через ИИ-чат

- Пользователь вводит задачу в свободной форме в чат
- ИИ распознаёт: название, дату, исполнителя из текста
- ИИ предлагает перейти на форму с предзаполненными данными
- Пользователь подтверждает и сохраняет задачу

## 6. Модель данных и статусы

### 6.1 Жизненный цикл задачи

| Статус | Цвет | Описание перехода |
|---|---|---|
| Новая | Синий | Задача создана, работа не начата |
| В работе | Оранжевый | Исполнитель принял задачу в работу |
| Выполнена | Зелёный | Задача завершена, статус подтверждён |
| Отложена | Красный | Задача приостановлена или заблокирована |

### 6.2 Структура объекта задачи (JSON)

```json
{
  "id": "number (timestamp)",
  "title": "string — наименование задачи",
  "date": "string — дата дедлайна (YYYY-MM-DD)",
  "priority": "normal | high | critical",
  "status": "new | prog | done | hold",
  "myEmail": "string — email владельца",
  "assignee": "string — ФИО исполнителя",
  "assigneeEmail": "string — email исполнителя",
  "assigneePhone": "string — телефон исполнителя",
  "created": "string — ISO дата создания"
}
```

## 7. Email-уведомления — детальное описание

### 7.1 Уведомление владельцу при создании

- **Тема:** «Задача создана: [Название задачи]»
- **Содержание:** название, дата дедлайна, приоритет, исполнитель (если назначен)
- Ссылка для просмотра задачи в системе

### 7.2 Уведомление исполнителю при назначении

- **Тема:** «Вам назначена задача: [Название задачи]»
- **Содержание:** описание задачи, дедлайн, контакт руководителя
- Инструкция: «Отправьте статус выполнения на [email руководителя] до [дата − 1 день]»

### 7.3 Напоминание исполнителю за 1 день

- **Тема:** «Завтра дедлайн по задаче: [Название задачи]»
- **Содержание:** напоминание отправить статус на email руководителя
- Срок отправки статуса: до конца текущего дня

### 7.4 Напоминание владельцу за 1 день

- **Тема:** «Напоминание: завтра срок по задаче [Название]»
- **Содержание:** сводка задачи, текущий статус, исполнитель

### 7.5 Уведомление в день дедлайна

- **Тема:** «Сегодня дедлайн: [Название задачи]»
- **Содержание:** финальное напоминание, статус задачи, кнопка смены статуса

## 9. Критерии приёмки (Acceptance Criteria)

### 9.1 Must Have (MVP)

- [ ] Форма создания задачи со всеми обязательными полями
- [ ] Обязательный вопрос о назначении исполнителя перед сохранением
- [ ] Email-уведомление владельцу при создании задачи
- [ ] Email-уведомление исполнителю при назначении
- [ ] Напоминание владельцу за 1 день и в день дедлайна
- [ ] Напоминание исполнителю за 1 день (отправить статус)
- [ ] Адаптивный интерфейс под мобильные устройства
- [ ] Список задач с фильтром по статусу
- [ ] Изменение статуса задачи

### 9.2 Should Have

- [ ] ИИ-чат с распознаванием задач из текста
- [ ] Интеграция с Google Calendar
- [ ] Уведомления с форматированием HTML-email
- [ ] Приоритизация задач (Обычный / Высокий / Критичный)

### 9.3 Nice to Have

- [ ] Push-уведомления в браузере
- [ ] Экспорт задач в CSV / Excel
- [ ] История изменений статусов
- [ ] Telegram-уведомления как альтернатива email
- [ ] Голосовой ввод задач

## 10. Риски и ограничения

| Риск | Уровень | Митигация |
|---|---|---|
| Email попадает в спам | Высокий | SPF/DKIM настройка домена, использование SendGrid |
| Недоступность Claude API | Средний | Fallback-режим без ИИ, кэш ответов |
| Несинхронизация календаря | Низкий | Экспорт .ics файла как резервный вариант |
| Переполнение email-квоты | Средний | Батчинг уведомлений, rate limiting |
| Потеря данных localStorage | Низкий | Серверное хранилище как основное |
